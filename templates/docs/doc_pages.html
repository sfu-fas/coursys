    Overview

    The course pages feature allows instructors to create course web pages within {{ CourSys }}. The pages can be created with simple wiki markup and access can be controlled as necessary.

    The pages created within the system can be used to create a simple course web site, or to augment an existing site.

    Full edit history is kept for each page, so you can see each version of the page that has been saved. Anyone who can edit the page can view the page history.

    Page Permissions

    Pages can have permission set for both viewing and editing. Each page can have separate permissions set, as needed. Some example uses of the page permissions:

    • Instructor can edit, anyone can view: classic course web page.
    • Only TAs can view: administrative course information, marking guidelines.
    • Only TAs can view, changed to students can view: assignment solutions released to students after due date.
    • Students can edit: shared course wiki page.

    Typically, only course instructors and TAs can create new pages. This setting can be changed per-offering by the system admin.

    Wiki Markup

    Pages are written with Wiki Creole Markup (list of Wiki Creole markup reference). Most Creole Additions also work.

    Here are some Wiki Creole basics. For full details, see the links above.

    Bold
    **bold text**
    Italic
    //italic text//
    Bulleted list
    * item one
* item two
    Numbered list
    # item one
# item two
    Headings
    == Section Heading ==
=== Subsection Heading ===
    Links
    [[AnotherPage]]
[[AnotherPage|Another page on this site]]
[[http://www.sfu.ca/|Another page on the web]]

    Creole Additions

    Some additions have been made to Creole markup that might be useful for course web pages…

    Due Dates

    You can automatically insert the due date of an activity like this:

    Assignment 1 is due <<duedate Assignment 1>>.

    As long as you spell the activity name correctly, the page will display the due date assigned for the activity and will update if you change it. If you'd like to include the date and the time, it is done like this:

    Assignment 2 is due at //exactly// <<duedatetime Assignment 2>>.

    Page Lists

    You can insert a list of all pages on your course site like this:

    <<pagelist>>

    Or, a list of all pages whose label starts with “Assign” (e.g. pages “Assignment 1”, “Assignment 2”) like this:

    <<pagelist Assign>>

    Formulas

    Formulas can be inserted into pages with TeX markup, using \(…\) to indicate inline formulas and \[…\] for display formulas. Processing is done with MathJax, so any TeX it supports should work.

    Solutions to the quadratic equation \(ax^2 + bx + c\) are

\[ x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}\,. \]

    You must check the “Math” box when editing the page to turn MathJax on. Formulas should be visible in any modern browser.

    Hilighted Code

    Code samples can be syntax-highlighted with SyntaxHighlighter. You only need to specify the language of the code in question. Here are examples of Python and C code that will be hilighted when displayed:

    {{{ [python]
for i in range(10):
    print i
}}}

{{{ [c]
for (int i=0; i<10; i++) {
  printf("%i\n", i);
}
}}}

    You can use any of the brush aliases known by SyntaxHighlighter to specify the language. Other brushes can be added on request.

    Files

    If you like, files can be upload to create a “page” instead of having an HTML page created with the wiki markup. This can be used to distribute PDFs or other documents.
